# Agency Ecosystem Showcase

This directory contains the first deliverable for Phase 33: "Agency Ecosystem Demonstration". It provides a functional simulation of a complex, multi-agency orchestration process.

## Purpose

The showcase demonstrates how a Root Orchestrator can break down a large project brief, spawn specialized child agencies with isolated parameters, and delegate tasks among them to achieve a unified goal. This directly maps to our **4-Pillar Vision**:

1.  **Company Context**: The `project_brief.md` serves as the high-level intent. Each spawned child agency is injected with a specific `context` (defined in `child_agency_configs/`), turning the generalized brief into specialized action.
2.  **SOP-as-Code**: The orchestration script rigidly follows a procedural logic—first spawning agencies, then distributing tasks across milestones—validating that complex business procedures can be fully automated.
3.  **Ghost Mode**: The execution simulates how the Simple-CLI agent operates autonomously without human intervention, breaking down goals, utilizing specialized sub-agents, and producing artifacts automatically.
4.  **Recursive Optimization**: While this is a simulated run, the artifacts generated by one agency (e.g., backend API schema) can serve as the direct context for another (e.g., frontend implementation), mirroring a continuously self-correcting and optimizing workflow.

## Running the Showcase

The orchestration script runs in a simulated environment, meaning it uses mocked LLM outputs and writes file artifacts locally instead of executing live code against real external APIs.

To execute the demonstration, run:

```bash
npx tsx demos/agency_ecosystem_showcase/orchestrate.ts
```

### What to Expect
1. The orchestrator will parse the `project_brief.md`.
2. It will utilize the Agency Spawning Protocol logic to create three specialized sub-agencies (Frontend, Backend, and Business Ops).
3. It will use the Federation Protocol to assign tasks to these agencies according to defined milestones.
4. You will see console logs outlining the step-by-step progress, and dummy artifacts will be generated in the `demos/agency_ecosystem_showcase/.agent/child_agencies/<uuid>/artifacts/` directories.

## Directory Structure

*   `project_brief.md`: The definition of the "TaskFlow Pro" SaaS project.
*   `orchestrate.ts`: The main TypeScript script managing the agency workflow.
*   `child_agency_configs/`: Contains JSON configuration files for the specialized child agencies, dictating their identity and allowed resources.
*   `.agent/`: (Generated upon run) The temporary isolated execution space for child agencies, storing policy files and generated artifacts.
